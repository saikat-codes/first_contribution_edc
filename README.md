# **🌿 WebD Task 01 — Git Practice**  
**EDC NIT Durgapur · Web Development Domain**  
## **📌 About This Task**  
This task was assigned by the **Web Development team at EDC NIT Durgapur** as part of the onboarding/skill-building process for new members.  
The goal was to get hands-on with **Git** — not just the theory, but actual terminal-based workflows that every developer uses on the job. Members were required to complete all  **22 problems on **[ **git-practice.com**](https://git-practice.com/ "https://git-practice.com/"), a browser-guided Git learning platform that walks you through real scenarios using a simulated terminal environment.  
## **🧠 What We Covered**  
Across the 22 problems, members worked through the full Git workflow — from the very basics to more advanced collaboration patterns:  
- Setting up and initialising repositories  
- Staging, committing, and writing meaningful commit messages  
- Branching, switching, and merging  
- Resolving merge conflicts  
- Working with remote repositories (push, pull, fetch)  
- Fork-based workflows — understanding origin vs upstream  
- Viewing history with git log and comparing changes with git diff  
- Rewriting history — git commit --amend, interactive rebase (git rebase -i)  
- Fixup commits and --autosquash  
- Pull Request workflows and code review practices  
- Amending commits to address review feedback and force-pushing to update PRs  
By the end, members had a solid practical understanding of how Git is actually used in collaborative software development.
